Submission Guide

A single route through the scattered registration, preparation, upload, status, and version pages.

Source: Submission Guidelines
1Confirm account and identityRequired

Submissions are accepted from registered authors. Each user should have one arXiv account, current affiliation, and a verified email address.

  • Usernames are unique and cannot be changed.
  • Claimed affiliation should be current in the conventional sense.
  • Misrepresenting identity or affiliation can lead to suspension.
2Check endorsement and eligibilityMay apply

New users or users submitting to a new category may need endorsement. Author self-submission is preferred.

  • Look for endorsers among authors of recent related arXiv papers.
  • Third-party submissions require prior authorization or approved proxy facilities.
  • Conference or journal proxies must classify papers individually, not by venue alone.
3Choose source formatRequired

TeX/LaTeX is the best archival choice when available. PDF is accepted only when it is machine readable and follows PDF requirements.

  • Do not submit DVI, PostScript, scanned documents, or PDF generated from TeX/LaTeX source.
  • HTML with JPEG, PNG, or GIF images is accepted for text submissions.
  • All figures must be included in the submitted source package.
4Package files and figuresRequired

Use portable file names, include required TeX support files, and remove files that do not belong in the archival source.

  • Allowed file-name characters: letters, numbers, underscore, plus, hyphen, period, comma, and equals sign.
  • File names are case sensitive. Match all internal references exactly.
  • Use ancillary files for large blocks of code, data, images, tables, movies, or animated GIFs.
5Enter metadataRequired

Title, authors, and abstract are required. Metadata supports search, announcements, RSS, indexing, and ownership records.

  • Metadata fields accept ASCII input; convert Unicode to TeX equivalents where needed.
  • Abstracts longer than 1920 characters are not accepted.
  • Use Comments for replacement notes, page counts, related URLs, and publication status before journal metadata exists.
6Upload, check, preview, submitRequired

Upload files, use Check Files, confirm the detected processor and top-level TeX file, preview the PDF, then submit.

  • If compilation fails, inspect the full log and fix the source before proceeding.
  • Mixed figure formats and case mismatches are common causes of processing failure.
  • Before final submission, carefully review the title, abstract, metadata, and generated PDF.

Format Checklist

Format problems should be fixed before the work is treated as ready for moderation.

Every submission should have

  • Title and authorship. Anonymous submissions are not accepted.
  • Complete references.
  • Machine-readable text.
  • Single-spaced body text, 10 to 14 point type, and at least 1 inch margins.
  • Publicly available repository links for referenced code or data sets.

Every submission should avoid

  • Line numbers, obstructive watermarks, highlighted text, margin notes, or referee remarks.
  • Advertisements, slides, posters, or long non-text blocks inside the article body.
  • Copyright statements that prohibit or impair arXiv redistribution.
  • Embedded JavaScript, movies, animated GIFs, or HTML inside a PDF.
  • Scanned documents or bitmapped whole-paper PDFs.

TeX and PDF calls

TeX source
Include needed style files, macros, figures, .bib or .bbl files, .ind files, and glossary outputs. Do not include generated output files or hidden directories.
PDF source
Use embedded fonts, standard structure, selectable text, and no JavaScript. Avoid Print to PDF when it rasterizes the document.
Figure formats
LaTeX processing uses PS/EPS. PDFLaTeX processing uses PDF, JPG, or PNG. arXiv does not convert figures during processing.

Metadata Checklist

Metadata is not decoration. It controls discovery, ownership, announcements, and version context.

Source: Metadata fields
Field Status Use it for Avoid
Title Required Concise paper title, with standard arXiv references where needed. All uppercase, Unicode characters, and opaque TeX macros.
Authors Required All authors in standard name order, with affiliations in parentheses when supplied. Honorifics, degrees, "et al.", commas inside names, fake affiliations, or AI tools as authors.
Abstract Required Short, readable summary for mailings and search. Limit is 1920 characters. The word "Abstract", unnecessary TeX spacing commands, long tables, or excessive line breaks.
Comments Optional Page counts, related URLs, replacement reasons, and publication status before journal metadata exists. Copyright statements or license text that conflicts with the selected arXiv license.
Journal-ref / DOI Optional Full publication information and a DOI resolving to another version, such as a journal article. URLs in Journal-ref or the arXiv-assigned DOI in the DOI field.

After Submission

Status, announcement, replacement, and version behavior in one place.

Incomplete

You can still edit. If not submitted or updated within 14 days, it is removed.

Processing

arXiv is processing the submission. Editing is unavailable during this usually short state.

Submitted

Final pre-announcement state. You may unsubmit to return it to incomplete status.

On hold

No immediate action is required unless arXiv contacts you. Do not delete and resubmit.

Announced

The arXiv identifier and DOI are assigned during announcement, not in advance.

Replacement

Public changes create a new version. Previous versions remain publicly available.

arxiv_new is a local reorganization prototype built from the saved source files in this folder. Original policy authority remains with the source documents and arXiv.
